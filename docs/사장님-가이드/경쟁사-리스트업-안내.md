# 경쟁사 리스트업 — 누가 우리 자리를 차지하고 있나

**요청(2026-07-23)**: "누락시킨거 보고 상위노출된 경쟁사 리스트업 (매번 갱신) / 제일 많이 보이는 애들 횟수 체크해서 갱신해주는 시스템"

## 무엇을 해주나

순위 검사기가 키워드를 검색할 때, 화면 위쪽 자리(AB·스마트블록·인기글)에 실제로 올라와 있는 글 목록을
같이 기록합니다. 우리 글은 빼고 남는 것이 **경쟁사**입니다. 매 실행마다 갱신됩니다.

- 수집 범위: **전 키워드** (우리가 누락된 키워드뿐 아니라 1등인 키워드까지)
- 우리 글 제외 기준: 시트에 있는 우리 링크 + 우리 카페 목록(CAFE_WHITELIST)
- 한 곳이 같은 키워드에 여러 글을 깔아도 **1회**로 셉니다(가장 높은 자리 기준)

## 어디서 보나

비공개 시트에 탭 2개가 생깁니다. 사장님 작업 탭은 건드리지 않습니다.

**경쟁사_랭킹** — 사장님이 볼 화면. 매 실행 전체 갱신.

| 이름 | 주체 | 종류 | 등장 횟수 | 노출 키워드 수 | 평균 순위 | 1위 횟수 | 우리가 놓친 키워드 수 | 최근 등장일 | 대표 URL |
|---|---|---|---|---|---|---|---|---|---|

- **이름** = 검색 결과에 뜬 그대로의 카페·블로그 이름 (예 "부산맘 카페 - 부산 맘카페 …").
- **등장 횟수** = (날짜 × 키워드) 조합. 같은 날 같은 키워드는 1회.
- **우리가 놓친 키워드 수** = 그 경쟁사가 보인 키워드 중, 우리 상태가 누락·미노출·삭제인 키워드 수.
  이 숫자가 큰 곳이 **우리 자리를 실제로 먹고 있는 곳**입니다.

**경쟁사_이력** — 원본 기록(날짜·키워드·구좌·순위·주체·이름·제목·URL). 최근 21일치 보관, 오래된 날짜는 자동 정리.
"누락 건만" 보고 싶으면 이 탭에서 `우리상태 = 누락` 으로 필터하면 됩니다.

## 비용·부담

- 검색 요청 추가 **없음**. 순위 검사할 때 이미 받아온 화면을 그대로 다시 읽습니다.
- 실행 시간 영향 거의 없음. 늘어나는 건 시트 쓰기뿐(하루 1회 적재 + 랭킹 갱신).
- 수집·기록이 실패해도 순위 검사와 시트 갱신은 그대로 진행됩니다.

## 지금 한계 (알고 계셔야 할 것)

1. **인기글 구좌**는 사람이 보는 화면과 봇이 보는 화면이 다를 때가 있습니다. 숫자는 봇 기준입니다.
2. 검색 결과에 이름이 안 실린 경우에는 이름 칸이 비고 주소로만 나옵니다(대부분은 이름이 실립니다).
3. 우리 카페 목록(`CAFE_WHITELIST_SLUGS`)이 비어 있으면 우리 글을 가려낼 수 없어 수집을 건너뜁니다.
4. 네이버 카페 주소가 새 형식(`ca-fe/cafes/숫자`)으로 나오고 이름도 안 실리면, 같은 카페가
   `카페#숫자` 로 따로 잡혀 **횟수가 갈릴 수 있습니다**. 이름이 같고 둘 다 카페일 때만 합칩니다 —
   이름만 같으면 합치던 방식은 서로 다른 블로그를 한 곳으로 만들어 없는 경쟁사를 1위로 올렸습니다.
   적게 세는 쪽이 지어내는 쪽보다 안전하다고 판단해 좁게 걸었습니다.
5. 랭킹 탭은 **상위 300곳**까지 표시합니다(그 아래는 이력 탭에 그대로 남습니다).

## 안전장치 (사고 나도 데이터가 안 날아가게)

- 기록 단위가 **(날짜 × 키워드)** 라, 네이버 차단으로 중간에 끊긴 실행은 자기가 돈 키워드만 갈아끼웁니다.
  그날 이미 쌓인 나머지 키워드 기록은 그대로 남습니다.
- 이력 적재가 실패하면 랭킹 탭은 **건드리지 않습니다**(빈 표로 덮어쓰지 않음). 지난 집계가 그대로 보입니다.
- 시트 행 삭제는 구간 단위 1회 호출로 처리합니다(예전 순위 검사기가 겪은 API 한도 초과 재발 방지).
