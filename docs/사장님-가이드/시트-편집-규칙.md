# 시트 편집 규칙 — 마케터 공유용 (T-M9.4, 2026-06-12)

> 2026-06-12 오전, 검사 도중 시트 편집 + 행 복사 때문에 74행이 한동안 "재검사필요"로 표시된 사건의 재발 방지 규칙입니다.
> 어기면 시트가 망가지지는 않지만(프로그램이 막아줌), 그 행들의 결과 표시가 다음 검사까지 늦어집니다.

## 규칙 1 — 검사 시간대에는 시트 편집을 피한다

검사 프로그램은 매일 **4번**, 시작 후 **약 70분간** 돌아갑니다:

| 검사 시작 (KST) | 편집 피해야 하는 시간 |
|---|---|
| 00:07 | 00:00 ~ 01:30 |
| 06:07 | 06:00 ~ 07:30 |
| 12:07 | 12:00 ~ 13:30 |
| 18:07 | 18:00 ~ 19:30 |

이 시간 동안 행 추가/삭제/정렬/링크 교체를 하면, 프로그램이 "행이 어긋났다"고 판단해 그 행 기록을 안전하게 건너뜁니다 → "재검사필요"로 표시됐다가 다음 검사에서 회복됩니다.

(시트 1행 P열의 "cron 갱신: ..." 시각이 최근 검사 완료 시각입니다.)

## 규칙 2 — 신규 행은 "빈 행에 직접 입력", 기존 행 복사 금지

새 작업 행을 만들 때:

- ✅ **빈 행에 A~J열만 직접 입력**: 작업일 / 작업자 / 유형 / 키워드 / MB / PC / 총합 / 작업아이디 / 카페·게시판 / 링크
- ❌ **기존 행을 복사해서 만들지 않기** — 눈에 안 보이는 숨김 칸(검사 기록)까지 따라 복사되어, 검사한 적 없는 행이 "검사한 척"하는 유령 값이 생깁니다.

## 규칙 3 — K열 이후는 시스템 전용

- K(노출영역) / L(통합탭 순위) / M(카페구좌순위) / O(지식인탭) + 숨겨진 Q~W열은 **프로그램이 자동으로 채우는 칸**입니다. 직접 입력/수정/복사하지 않습니다.
- 메모가 필요하면 K열이 아니라 별도 메모 칸(N열 등 사용 중인 칸)에 적습니다.

## "재검사필요"가 보이면?

- 고장이 아닙니다. **"키워드나 링크가 바뀌어서 다음 검사 때 다시 확인할 행"**이라는 표시입니다.
- 6시간 안에 다음 검사가 자동으로 실제 값을 채웁니다. 기다리면 됩니다.
- 급하면 사장님께 요청 → GitHub Actions에서 `recheck_stale_only=true` 수동 실행으로 그 행들만 즉시 재검사할 수 있습니다.

## 시트 범위 보호 설정 (사장님 1회 작업, 5분)

마케터가 실수로 시스템 칸을 건드리지 못하게 경고를 켭니다:

1. 시트에서 **K열~W열 전체 선택** (열 머리글 K 클릭 → Shift 누르고 W 클릭)
2. 우클릭 → **"범위 보호"** (Protect range)
3. "수정 권한" → **"이 범위를 수정할 때 경고 표시"** 선택 → 저장
4. 탭(샴푸 카외 / 바디워시 카외 / 두드러기 카외)마다 반복

이렇게 하면 마케터가 K열 이후를 수정하려 할 때 "정말 수정하시겠습니까?" 경고가 떠서 실수를 막아줍니다. (검사 프로그램은 서비스 계정이라 영향 없이 계속 기록합니다 — 경고 모드는 외부 프로그램 기록을 막지 않습니다.)
