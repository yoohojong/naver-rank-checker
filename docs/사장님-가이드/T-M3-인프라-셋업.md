# 사장님 인프라 셋업 가이드 (T-M3.1, T-M3.2, T-M3.3)

**대상**: 사장님. 코드 수정 X, 클릭만.
**예상 소요**: 30~40분 (한 번에 또는 나눠서)
**전제**: 본인 Google 계정 + GitHub 계정 있음.

---

## ✅ 작업 순서 (3개)

1. **T-M3.1**: GCP 프로젝트 + Sheets API + 서비스 계정 발급 (15분)
2. **T-M3.2**: 사장님 시트에 서비스 계정 이메일 공유 (1분)
3. **T-M3.3**: GitHub 공개 저장소 생성 (5분)

---

## T-M3.1 — GCP 프로젝트 + 서비스 계정 발급

### 1️⃣ GCP 콘솔 접속

→ https://console.cloud.google.com 접속, 본인 Google 계정 로그인.
→ 처음이면 약관 동의 화면 나옴. 체크 + 동의 클릭.

### 2️⃣ 새 프로젝트 만들기

→ 화면 왼쪽 위 "프로젝트 선택" (또는 가운데에 큰 "프로젝트 만들기" 버튼) 클릭.
→ "**새 프로젝트**" 클릭.
→ 프로젝트 이름: `naver-rank-checker` (또는 원하는 이름).
→ 위치/조직: 그냥 두기.
→ "**만들기**" 클릭. 약 1분 대기 (오른쪽 위 알림 나옴).
→ 알림 떴으면 "**프로젝트 선택**" 클릭해서 새 프로젝트로 진입.

### 3️⃣ Google Sheets API 활성화

→ 화면 왼쪽 메뉴 (햄버거 ≡) 클릭 → "**API 및 서비스**" → "**라이브러리**".
→ 검색창에 `Google Sheets API` 입력 → 첫 번째 결과 클릭.
→ "**사용**" 버튼 클릭. 완료 (1분).
→ 같은 방법으로 `Google Drive API` 도 검색 + 사용 (gspread 가 시트 access 위해 필요).

### 4️⃣ 서비스 계정 만들기

→ 화면 왼쪽 메뉴 → "**API 및 서비스**" → "**사용자 인증 정보**".
→ 위쪽 "**+ 사용자 인증 정보 만들기**" → "**서비스 계정**" 선택.
→ 이름: `naver-rank-bot` (또는 원하는 이름).
→ "**만들고 계속하기**" → 역할 선택 화면 나오면 그냥 "**계속**" → "**완료**".

### 5️⃣ JSON 키 다운로드

→ 방금 만든 서비스 계정 (목록에 `naver-rank-bot@<프로젝트id>.iam.gserviceaccount.com` 형태) 클릭.
→ 위쪽 "**키**" 탭 → "**키 추가**" → "**새 키 만들기**".
→ 키 유형: **JSON** 체크 → "**만들기**".
→ JSON 파일이 자동으로 PC 에 다운로드됨. **이 파일을 안전한 곳에 저장**.
→ 파일 안의 `client_email` 값 (이메일 형식) 을 메모장에 복사해둠. T-M3.2 에서 사용.

⚠️ JSON 파일은 비밀번호 같은 것. **누구에게도 공유 X**. GitHub 에 올리지도 X.

---

## T-M3.2 — 사장님 시트에 서비스 계정 공유

### 1️⃣ 사장님 시트 열기

→ Google Sheets 에서 사장님 운영 중인 시트 열기.

### 2️⃣ 공유

→ 우측 위 "**공유**" 버튼 클릭.
→ 사람 추가란에 **T-M3.1 step 5 에서 메모한 client_email** 붙여넣기 (예: `naver-rank-bot@xxx.iam.gserviceaccount.com`).
→ 권한: **편집자**.
→ "보내기" 버튼 클릭 (이메일 알림 끄려면 체크박스 해제).

→ 완료. 시트가 서비스 계정에서 접근 가능해짐.

---

## T-M3.3 — GitHub 공개 저장소 생성

### 1️⃣ GitHub 접속

→ https://github.com 본인 계정 로그인.

### 2️⃣ 새 저장소 만들기

→ 우측 위 "+" 아이콘 → "**New repository**" 클릭.
→ Repository name: `naver-rank-checker`.
→ Description: (선택) `네이버 키워드 상위노출 자동 체크` 같이 한 줄.
→ **Public** 선택 (⚠️ 무료 GitHub Actions 사용 위해 필수).
→ "Add a README file" 체크 (있어도 됨, Claude 가 나중에 덮어씀).
→ "**Create repository**" 클릭.

### 3️⃣ 저장소 URL 복사

→ 만들어진 저장소 페이지에서 위쪽 URL (예: `https://github.com/<사장님id>/naver-rank-checker`) 복사.
→ Claude 에게 알려주기 (T-M3.4 에서 로컬 git init + 원격 연결).
