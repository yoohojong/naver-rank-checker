# Spec: naver-rank-checker

**Last updated**: 2026-05-05
**Status**: brainstorming 5 섹션 완료, 셀프 리뷰 통과, 사장님 검토 대기 중

---

## 1. 목적

네이버 검색에서 사장님(유호종)이 발행한 카페 글의 노출 영역과 순위를 주기적으로 자동 체크하고, 결과를 사장님이 운영 중인 Google Sheets(분야별 탭)에 자동 기록한다.

## 2. 배경

- 2026-03 프로그램88에 33만원 외주로 동일 기능 PC 설치형 프로그램 받음
- 외주본 문제:
  - 백신이 PHP/XAMPP 모듈(php8ts.dll) 삭제 → 작동 불능
  - 네이버 코드 변경 시 5만원 + 수일 대기
  - naver.me 단축 URL 인식 안 됨
  - 비실계 발행글 누락
  - 가장 큰 미충족: **Google Sheets 자동 연동 + 매시간 갱신** 부재
- 자체 제작으로 안정성 + Sheets 자동화 + 무료 운영 모두 확보 목표

## 3. 사용자 페르소나

- 비개발자 (사장님 본인)
- 코드 직접 수정 안 함, 결과(시트)만 봄
- Google Sheets 운영 중: 분야별 탭, 각 탭은 약 15개 컬럼 (작업일·작업자·유형·키워드·MB·PC·총합·작업아이디·카페/게시판·링크·노출영역·통합탭순위·카페구좌순위·블로그구좌순위·지식인탭)
- 1000개+ 키워드/URL 쌍 추적 (확장 가능성 무제한)

## 4. 핵심 요구사항

### 4.1 입력 (사용자가 시트에 직접 채움)
- 작업일, 작업자
- 키워드
- 검색량 (모바일 / PC / 총합) — 수동
- 작업아이디 (네이버 카페 ID)
- 카페/게시판 (반자동 — 처음 fetch 후 매핑 시트에 저장)
- 카페 글 링크 (URL)

### 4.2 출력 (프로그램이 자동 채움) — 사장님 기존 K~O 컬럼만
- **유형 (C)**: 키워드 검색 시 탭/블록 노출 순서 (예: "AB,인기글,스마트블록")
- **노출영역 (K)**: 본인 URL 상태를 모두 담는 단일 컬럼
  - `AB` / `스마트블록` / `인기글` — 현재 정상 노출
  - `미노출` — 추적 시작 이래 한 번도 노출된 적 없음 (URL은 살아있음)
  - `노출중지` — 이전엔 노출됐는데 떨어짐 (상태 전환 자동 감지: 시트의 이전 K 값과 비교)
  - `삭제됨` / `비공개` — URL 자체 죽음
  - `실패` — 처리 중 에러 (다음 cron에 자동 재시도)
  - **상태 전환 규칙** (사이드 이펙트 없는 자동 감지):
    - 이전 K = 노출 값 (`AB` / `스마트블록` / `인기글`) + 지금 검색 0 → `노출중지`
    - 이전 K = `노출중지` + 지금 여전히 검색 0 → `노출중지` 유지
    - 이전 K = `노출중지` + 지금 다시 노출 → 노출 값으로 복귀
    - 이전 K = `미노출` + 지금 검색 0 → `미노출` 유지 (한 번도 안 됐으면 계속 미노출)
- **노출여부(통합탭 순위) (L)**: AB 통합 긴 리스트 안에서의 절대 순위 — 사장님 시트 실 헤더명 (2026-05-07 확인)
- **노출여부(카페구좌순위) (M)**: AB 리스트 안 카페 항목들 중 순위 — 사장님 실 헤더
- **노출여부(블로그구좌순위) (N)**: AB 리스트 안 블로그 항목들 중 순위 — 사장님 실 헤더. **2026-05-08 사장님 명시: "삭제할 예정" → 우리 시스템 안 채움 (write 흐름에서 자동 skip, 사장님 컬럼 삭제 후 자연 해소)**.
- **지식인탭 (O)**: 지식iN 탭 노출 시 "O"

**시트에 안 남기는 것** (사장님 요청):
- 별도 알림 메시지, 헬스 보고, 실패 카운트 등 — GitHub Actions logs만
- 시트는 데이터만, 깔끔하게 유지

### 4.3 자동화 정책
- **주기**: 6시간마다 4회/일 (GitHub Actions cron `0 */6 * * *`)
- **스케일**: 1000개+ (무제한) 키워드/URL
- **신규 시트 탭**: `카페매핑` (슬러그 → 카페정식명 → 사용자축약명)
- **인기글 별도 로직**: 통합탭(AB)과 다른 파싱 경로

## 5. 기술 결정 (확정)

| 항목 | 결정 | 근거 |
|------|------|------|
| 실행 환경 | GitHub Actions 공개 저장소 | 진짜 무료, 무제한, 카드 등록 불필요 |
| 언어 | Python | Sheets API + 크롤링 라이브러리 풍부, GitHub Actions 친화 |
| 코드 구조 | Modular B (6 모듈) | 99%+ 견고성 + 무제한 스케일 + 헬스체크 분리 |
| Sheets 매핑 | 헤더 이름 기반 | 열 이동/추가에 강건 |
| 시트 구조 | 분야별 탭 순회 | 사장님 기존 사용 방식 |
| 검증 방식 | 캐시 (카페 매핑, 결과) + 재시도 큐 | 99%+ 성공률 |

## 6. 모듈 책임 (Modular B)

```
src/
├── crawler.py    네이버 검색 페이지 fetch (HTTP requests, 슬로우다운, User-Agent 회전)
├── parser.py     AB / 스마트블록 / 인기글 / 지식인 파싱 분기
├── sheets.py     Google Sheets I/O (서비스 계정), 헤더 기반 매핑
├── cache.py      카페 매핑 시트, 결과 캐시
├── retry.py      재시도 큐 (1차 실패 → 슬로우다운 강화 후 1회 더, 5회 추적 X — 매 cron 독립)
├── health.py     파싱 성공률 모니터링, 네이버 코드 변경 감지 (출력은 GitHub Actions logs)
└── main.py       오케스트레이션 (위 모듈 조합)
```

## 7. 엣지케이스 처리

모든 상태는 **K(노출영역) 컬럼 단일 enum**으로 표현. 별도 비고/메시지 컬럼 없음.

| 상황 | K 컬럼 값 | 부가 동작 |
|------|----------|---------|
| 정상 노출 (AB) | `AB` | L/M/N에 순위 |
| 정상 노출 (스마트블록) | `스마트블록` | (해당 없으면 L/M/N 빈 칸) |
| 정상 노출 (인기글) | `인기글` | (인기글 별도 로직, M 또는 별도 처리) |
| 검색 0 + URL 살아있음 + 이전엔 한 번도 노출 X | `미노출` | L/M/N 빈 칸 |
| 검색 0 + URL 살아있음 + 이전엔 노출됐었음 | `노출중지` ⭐ | L/M/N 빈 칸 (이전 순위는 안 보존) |
| 카페 글 삭제 (404) | `삭제됨` | L/M/N 빈 칸 |
| 카페 글 비공개 (로그인 벽) | `비공개` | L/M/N 빈 칸 |
| 처리 중 에러 | `실패` | 다음 cron 자동 재시도 |
| naver.me 단축 URL | (자동 해석 후 위 흐름) | crawler가 풀 URL로 변환 |
| 같은 URL 여러 키워드 | (각 행 독립 처리) | 영향 없음 |
| 네이버 일시 차단 | (자동 슬로우다운 + 재시도 큐) | retry.py 처리 |
| GitHub Actions 환경 실패 | — | 다음 cron 자동 시작 |
| 네이버 HTML 구조 변경 | (모든 행 `실패` 가능) | health.py가 logs에 알림, 사장님 → Claude 재요청 |

## 8. 비기능 요구사항

- **비용**: $0 (카드 등록 불필요)
- **유지보수**: 사장님이 코드 수정 안 함, Claude(Anthropic 모델)로 재요청
- **성공률**: 매 cron 사이클 99%+ 목표 (재시도 1회 포함). 100% 보장은 기술적으로 불가 (무료 단일 IP 제약)
- **응답 정확성**: 외주본 동등 이상
- **운영 가시성**: 시트는 데이터만, 운영 정보(처리 시간, 차단 의심, 코드 변경 알림 등)는 GitHub Actions logs
